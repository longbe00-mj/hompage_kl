# 포트폴리오 카드 롤링 설계서

**작성일:** 2026-05-10  
**프로젝트:** THE KL LINK GROUP 홈페이지  
**변경 범위:** 포트폴리오 섹션 - 카드 롤링/캐러셀 구현

---

## 1. 개요

포트폴리오 섹션의 "진행 프로젝트 & 포트폴리오" 영역을 카드 롤링 방식으로 변경합니다. 한 번에 2개 카드가 보이면서 자동으로 롤링되는 캐러셀 기능을 구현합니다.

---

## 2. 핵심 요구사항

### 2.1 기능
- **카드 표시:** 데스크톱에서 2개 카드 동시 표시, 모바일에서 1개 카드 표시
- **자동 롤링:** 5초 간격으로 자동 롤링 (무한 루프)
- **네비게이션:** 좌우 화살표 버튼 + 하단 도트 인디케이터
- **상호작용:** 마우스 오버 시 자동 롤링 일시 중지, 떠날 때 재개
- **터치 지원:** 모바일에서 스와이프로 카드 이동 가능 (Embla 기본 지원)

### 2.2 반응형 디자인
| 화면 크기 | 동시 표시 카드 수 | 동작 |
|----------|------------------|------|
| 640px 이상 (md) | 2개 | 자동 롤링 + 네비게이션 |
| 640px 미만 | 1개 | 자동 롤링 + 네비게이션 |

### 2.3 기술 스택
- **라이브러리:** Embla Carousel
- **스타일링:** Tailwind CSS
- **애니메이션:** Embla 기본 애니메이션 + 커스텀 CSS

## 3. 상세 설계

### 3.1 컴포넌트 구조

```
PortfolioSection
  ├─ 기존 좌측/우측 레이아웃 유지
  ├─ [새로 추가] PortfolioCarousel
  │  ├─ Embla 컨테이너
  │  ├─ 캐러셀 슬라이드 (portfolioItems.map)
  │  ├─ 좌측 화살표 버튼
  │  ├─ 우측 화살표 버튼
  │  └─ 하단 도트 인디케이터
  └─ 기타 섹션 유지
```

### 3.2 데이터 구조

기존 `portfolioItems` 배열은 그대로 사용:
```typescript
const portfolioItems = [
  { id: 1, category: '..', title: '..', ... },
  { id: 2, category: '..', title: '..', ... },
  // ... 현재 7개 항목
];
```

### 3.3 PortfolioCarousel 컴포넌트

**입력:**
- `items: PortfolioItem[]` - 표시할 포트폴리오 항목 배열

**기능:**
- Embla Carousel 초기화 (2개 슬라이드, 모바일 1개)
- 5초 자동 롤링 (autoplay)
- 마우스 오버 시 일시 중지, 떠날 때 재개
- 좌우 화살표 클릭 시 슬라이드 이동
- 도트 클릭 시 해당 슬라이드로 이동
- 반응형 break point (md: 2개, 기본: 1개)

**출력:**
캐러셀 UI (카드, 네비게이션, 인디케이터)

### 3.4 Embla 설정

```typescript
const emblaOptions = {
  align: 'start',
  loop: true,
  skipSnaps: false,
  inViewThreshold: 0.7,
};

const plugins = [
  EmblaCarouselAutoplay({
    delay: 5000,
    stopOnInteraction: true, // 상호작용 시 자동 중지
  }),
];
```

### 3.5 스타일링

**캐러셀 컨테이너:**
- `overflow-hidden rounded-2xl`
- `w-full` (반응형 너비)

**슬라이드:**
- `flex-shrink-0` (크기 유지)
- `md:basis-1/2` (데스크톱: 50%)
- `basis-full` (모바일: 100%)
- 좌우 padding으로 카드 간 간격 (gap-4)

**네비게이션 버튼:**
- 위치: 캐러셀 양쪽 (left/right)
- 스타일: 기존 홈페이지 버튼 스타일과 통일
- 호버 시: 배경색 변경

**도트 인디케이터:**
- 위치: 캐러셀 하단
- 활성 도트: 파란색 (text-blue-600)
- 비활성 도트: 회색 (text-gray-400)
- 클릭 가능: 해당 슬라이드로 이동

### 3.6 파일 변경 계획

**신규 파일:**
- `src/components/sections/PortfolioCarousel.tsx` - 캐러셀 컴포넌트 (약 200줄)

**수정 파일:**
- `src/components/sections/PortfolioSection.tsx`
  - portfolioItems 배열을 PortfolioCarousel 컴포넌트로 전달
  - 기존 그리드 렌더링 부분 제거
  - Embla import 추가

**의존성:**
- `embla-carousel-react` npm 패키지 추가
- `embla-carousel-autoplay` npm 패키지 추가

## 4. 구현 세부사항

### 4.1 자동 롤링 동작

- **시작:** 페이지 로드 시 자동으로 시작
- **중지:** 
  - 마우스 오버
  - 화살표 또는 도트 클릭
  - 터치/스와이프
- **재개:** 마우스 떠난 후 5초 이후 또는 자동 체크

### 4.2 반응형 브레이크포인트

- `md` (768px) 기준
- 768px 이상: 2개 카드 + 간격 (gap-4)
- 768px 미만: 1개 카드 전체 너비

### 4.3 접근성

- 버튼에 `aria-label` 추가
- 도트에 `role="button"` 및 `tabindex`
- 키보드 네비게이션 지원 (Embla 기본)

### 4.4 성능

- Embla는 경량 라이브러리 (약 3KB)
- 번들 크기 영향 최소
- requestAnimationFrame 기반 애니메이션 (60fps)

## 5. 데이터 흐름

```
PortfolioSection
  ↓
portfolioItems 배열 (현재 7개)
  ↓
PortfolioCarousel 컴포넌트
  ├─ Embla 초기화
  ├─ 자동 롤링 플러그인 활성화
  ├─ 슬라이드 렌더링 (map)
  ├─ 네비게이션 버튼 바인딩
  └─ 도트 인디케이터 바인딩
  ↓
화면 표시 (2개 카드 롤링)
```

## 7. 기술 결정

| 결정 | 선택 | 이유 |
|------|------|------|
| 캐러셀 라이브러리 | Embla Carousel | Next.js 호환성, 경량, 자유도 높음 |
| 자동 롤링 간격 | 5초 | 충분한 읽기 시간 제공 |
| 카드 수 (데스크톱) | 2개 | 사용자의 요청 |
| 카드 수 (모바일) | 1개 | 화면 크기 고려 |
| 네비게이션 | 화살표 + 도트 | 사용성 극대화 |
| 루프 | 무한 | 연속적인 롤링 경험 |

---

## 8. 주의사항

- Embla는 `use client` 지시문 필요 (클라이언트 컴포넌트)
- 자동 롤링 플러그인은 `stopOnInteraction: true` 설정으로 사용자 상호작용 후 재시작
- 모바일에서도 터치 지원이 자동으로 활성화됨 (추가 설정 불필요)
- 도트 클릭 시 정확한 인덱스로 이동하기 위해 상태 관리 필요
